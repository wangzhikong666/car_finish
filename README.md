# 智能康复理疗机器人系统 (Intelligent Massage Robot System)

> **所属模块**：核心算法与综合调度控制包 (ROS Package: `car_finish`)
> **项目架构**：云-边-端协同 (Cloud-Edge-Device) / 具身智能 (Embodied AI)
> **当前版本**：v1.0 (敏捷开发集成版)

---

## 📖 1. 项目简介
本项目旨在打造一款基于机器视觉与车联网架构的智能康复理疗移动机器人。本代码库（`car_finish`）运行于边缘计算节点（如广和通算力板卡），承载了系统的核心控制中枢功能。系统深度集成了 **YOLO 视觉感知**、**底盘动态寻迹**、**五轴机械臂关节空间力控**、**多源传感器冗余防撞** 以及 **MQTT 跨网关协同** 等核心模块。

---

## 📂 2. 目录结构与核心功能说明

本功能包采用模块化解耦设计，各子系统通过 ROS 话题（Topic）进行高频异步通讯。核心文件说明如下：

*   **`massage.launch`** 
    *   **系统总启动文件。** 通过该脚本可一键并行拉起底层底盘驱动与所有核心控制节点，实现复杂多进程的自动化编排与调度。
*   **`turn_on_wheeltec_robot.launch` (被包含启动)**
    *   **底层底盘驱动启动文件。** 负责拉起小车的硬件通讯，发布里程计 (`/odom`) 等底层传感器数据，为上层导航提供物理基础。
*   **`inference.py`** 
    *   **视觉感知推理节点。** 调用前端摄像头画面，加载 `best.pt` 权重进行人体目标（如腿部）的高帧率特征提取与坐标偏移解算。
*   **`massage_nav.py`** 
    *   **底盘动态寻迹导航节点。** 将视觉偏移量转化为底盘的偏航角速度与线速度，实现自动逼近并在最佳理疗距离（0.3m）处平滑驻车。
*   **`my_massage_core.py`** 
    *   **机械臂核心控制与调度节点。** 采用“关节空间降维控制”策略彻底消除运动学逆解盲区。内置时间窗口防抖与 10 秒强制冷静期安全机制。
*   **`iot_mqtt_bridge.py`** 
    *   **车联网云边协同节点。** 建立与公网 MQTT 服务器的低延迟长连接，监听移动端下发的最高优先级启停指令，构筑跨网段安全熔断机制。
*   **`fsr402_driver.py`** 
    *   **压力传感器力控反馈节点。** 实时采集末端物理反作用力，支撑柔性力控闭环与肌肉僵硬度健康评估算法。
*   **`ultrasonic_driver.py`** 
    *   **超声波防撞与定距节点。** 构筑底层物理防撞安全边界，补足纯视觉方案在近距离作业下的感知盲区。
*   **`best.pt`** 
    *   **YOLO 视觉模型权重文件。** 经过特定理疗场景数据集（Custom Dataset）微调后的神经网络参数。

## 🚀 3. 快速启动 (Quick Start)

**步骤 1：环境初始化**
确保边缘计算平台已安装 ROS 环境（如 Noetic），并安装必要的 Python 依赖包：
```bash
pip install paho-mqtt rospy
```

**步骤 2：编译工作空间**
将本功能包放置在 `catkin_ws/src` 目录下，并在工作空间根目录执行编译：
```bash
cd ~/catkin_ws
catkin_make
source devel/setup.bash
```

**步骤 3：一键启动系统**
运行系统总调度脚本，一键启动包含底盘驱动在内的全系列服务：
```bash
roslaunch car_finish massage.launch
```
